# Fullstack AI-Contact Form 🚀

Проект представляет собой современное fullstack-приложение для отправки заявок через форму обратной связи. Главная особенность проекта — интеграция бэкенда с искусственным интеллектом (LLM) для автоматического улучшения и анализа пользовательского текста в реальном времени, а также полностью контейнеризированная архитектура для быстрой развертки в облачных PaaS-платформах.

---

## 📂 Структура репозитория

```text
├── backend/            # API-сервер на Node.js (TypeScript)
├── frontend/           # Клиентская часть (Vite + SCSS + TS)
├── .env.example        # Пример глобальных переменных окружения
├── .gitignore          # Игнорируемые файлы для Git
├── Dockerfile          # Комбинированный продакшн Dockerfile для Amvera
└── docker-compose.yml  # Локальная конфигурация Docker-оркестрации

```

## 🛠 Технологический стек

### Frontend

* **Библиотека/Сборщик:** Vite + TypeScript
* **Стилизация:** SCSS (использование модульной структуры `_variables.scss`, `_mixins.scss` и БЭМ-архитектуры)
* **Валидация:** Кастомный JS-валидатор с маской ввода номера телефона.

### Backend

* **Платформа:** Node.js (TypeScript)
* **Фреймворк:** Express.js
* **Управление окружением:** `dotenv` (с ручной строгой валидацией и парсингом типов в `env.ts`)
* **Отправка почты:** `nodemailer` (поддержка динамического переключения стратегий SMTP-шлюза)
* **Интеграция с ИИ:** Официальный SDK [`openai`](https://github.com/openai/openai-node) для отправки запросов к языковым моделям (LLM).
* Реализован изолированный сервис `AIService` для обработки текста.
* Настроен гибкий `baseURL`, что позволяет переключаться между оригинальным API OpenAI и сторонними эндпоинтами (например, OpenCode API).



### Infrastructure & DevOps

* **Локально (Docker Compose):** Раздельные контейнеры: Node.js (API на порту 5000) и легковесный сервер Nginx (порт 80), проксирующий запросы `/api/*`.
* **В облаке (Amvera):** Комбинированная сборка в едином контейнере на базе образа `hoosin/alpine-nginx-nodejs`, адаптированная под требования платформы слушать порт 80.

## 📝 Реализация формы обратной связи

Форма обратной связи реализует полноценный асинхронный UX и защищена на нескольких уровнях:

1. **Комплексная валидация (Фронтенд):**
* Написан единый цикл валидации, проверяющий как стандартные атрибуты HTML5 (`validity.valid`), так и кастомные правила.
* Реализована **динамическая маска ввода номера телефона** на чистом TypeScript. Любые попытки ввести буквы блокируются, а цифры автоматически форматируются в вид `+7 (999) 999-99-99`.
* Кнопка отправки защищена от double-click с помощью лоадера (`.form--loading`).


2. **Стилизация под автозаполнение:** Перебиты назойливые дефолтные стили браузеров (Chromium `:-webkit-autofill`), из-за которых поля ввода окрашивались в белый цвет. С помощью SCSS фон автозаполнения бесшовно синхронизирован с общей темной темой веб-интерфейса.
3. **Бизнес-логика отправки (Бэкенд):**
* Реализовано гибкое подключение к почте: через встроенный идентификатор `SMTP_SERVICE` (для Gmail, Mail.ru и др.) либо через кастомный `SMTP_HOST` / `SMTP_PORT` (для корпоративной почты).
* Осуществляется параллельная отправка писем: основное уведомление улетает владельцу сайта, а копия-подтверждение — пользователю.

## 🤖 Использование AI-инструментов

### Какие инструменты использовались:

* **OpenAI API / OpenCode:** Интеграция в бэкенд для интеллектуальной обработки текстовых полей.
* **AI-ассистент:** Помощь в генерации Docker-конфигураций, рефакторинге TypeScript-кода и адаптации SCSS-стилей.

### Что делалось с помощью ИИ:

1. **Интеграция функции «AI Улучшайзер»:** На фронтенде добавлена кнопка, отправляющая сырой текст комментария пользователя на бэкенд. Нейросеть исправляет орфографию, стилистику и делает тон текста более профессиональным, строго сохраняя при этом исходный смысл и язык пользователя.
2. **Архитектурные наброски:** Генерация базовых шаблонов `Dockerfile` и `docker-compose.yml`.

### Что пришлось исправлять вручную (Ошибки ИИ):

* **Валидация окружения:** ИИ предлагал проверять переменные через тяжелую библиотеку `Zod`. Было принято решение отказаться от лишней зависимости и написать надежную проверку типов и присутствия ключей в `env.ts` вручную на чистом TS.
* **Логика Docker Compose:** Первоначально ИИ сгенерировал проброс переменных без учета контекста путей файлов `.env`, из-за чего контейнер бэкенда падал с ошибкой отсутствия ключей. Ошибка исправлена ручной настройкой путей и переносом `.env` в корень для корректного маппинга переменных `${VARIABLE}`.
* **Строгая типизация в TS:** Исправлены типы данных в конфигурации `nodemailer` (ИИ сгенерировал порты и флаги безопасности в виде `string`, тогда как требовались `number` и `boolean`). Код был защищен объединением интерфейсов `TransportServiceConfig | TransportHostConfig` и явным приведением типов (`as`).
* **CSS Специфичность:** Системные стили автозаполнения браузера использовали флаг `!important`. Сгенерированные ИИ обычные стили не перебивали поведение псевдокласса `:-webkit-autofill`, поэтому оно исправлялось вручную через хак с внутренней тенью `box-shadow ... inset`.

## 🚀 Инструкция по запуску

### 1. Подготовка переменных окружения

Создайте файл **`.env`** в корневой директории проекта и заполните его:

```ini
PORT=5000
NODE_ENV=production

# === Настройки почты ===
# Вариант А: популярный сервис (mailru, gmail и т.д.)
SMTP_SERVICE=mailru
# Вариант Б: кастомный хост (раскомментировать, если нет SMTP_SERVICE)
# SMTP_HOST=smtp.mail.ru
# SMTP_PORT=465

SMTP_USER=your_email@mail.ru
SMTP_PASS=your_app_password
MAIL_TO=your_email@mail.ru

# === Настройки ИИ ===
OPENAI_API_KEY=sk-or-your-key-something...
OPENAI_API_MODEL=gpt-4o-mini

# Базовый URL API (Заполняйте ТОЛЬКО если используете сторонний сервис/прокси вместо OpenAI, например OpenCode)
# Если используете оригинальный OpenAI — оставьте строку закомментированной
# OPENAI_BASE_URL=[https://api.openai.com/v1](https://api.openai.com/v1)

```

### 2. Локальный запуск (Docker Compose)

Для локальной разработки и тестирования многоконтейнерной среды выполните команду в корне проекта:

* **В Bash / Linux:** `docker-compose up --build`
* **В PowerShell / Windows:** `docker compose up --build`

Приложение станет доступно по адресам:

* **Фронтенд (Nginx):** `http://localhost:8080` (перенаправляет API запросы на бэкенд)
* **Бэкенд API:** `http://localhost:5000`

### 3. Развертывание в облаке (Amvera)

Платформа Amvera использует одиночный контейнер, который обязан слушать порт 80. Для этого в корне проекта подготовлен комбинированный `Dockerfile` на базе образа `hoosin/alpine-nginx-nodejs`.

1. Привяжите свой Git-репозиторий или сделайте Push в репозиторий Amvera.
2. В личном кабинете Amvera перейдите во вкладку **«Переменные»** и укажите все ключи из файла `.env` (кроме портов).
3. Платформа автоматически обнаружит `Dockerfile`, соберет статику фронтенда, скомпилирует бэкенд и запустит проект на едином порту 80.
